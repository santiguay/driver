4. 安装与卸载

4.1 运行方法
    请遵照“EPSON OPOS ADK使用手册安装程序手册”（Installer_SC.pdf）中的步骤
    进行安装与卸载。

4.2 完整数据包与单位数据包
    本产品包含两种类型的数据包。
     - 完整数据包：为所有设备提供驱动程序
     - 单位数据包：
       为指定的POSPrinter以及能与该打印机连接的多分支设备提供驱动程序
       为单位数据包提供一个基础完整数据包版本。
       单位数据包可以与其它完整数据包以及单位数据包共存。
    有关与其它数据包共存的情况，请注意以下事项：
     - 如果已安装同一版本或更新版本的完整数据包，则不需安装单位数据包。
     - 当卸载单位数据包时，所有OPOS数据包都将被卸载。
     - 以下数据包的最新版本将显示在“添加与删除程序”与SetupPOS标题栏中：
        - 完整数据包版本
        - 单位数据包所依据的完整数据包版本
    注意：
      在已安装了单个包的环境下安装本产品时，单个包将被符合UPOS1.13的驱动程序
      置换。

5.2.1 安装
    1) 当使用键盘执行安装程序时，后台屏幕在出现错误消息
       或警告后可能会切换到无法使用键盘执行的屏幕。
       在这种情况下，请同时按下<Alt + F6>按钮并切换该屏幕。
    2) 如果某个不被支持的设备已在要安装该版本的环境中注册，
       则必须删除该设备的注册信息。
       请在安装该版本前删除该注册信息。如果未删除上述注册信息即安装该版本，
       则该注册信息将会在安装过程中被删除。
    3) 在本产品的以前版本（2.68版本-2.70版本）上进行无提示覆盖安装时，
       请重新启动系统。

5.2.2 安装附带的先前的版本（1.60版本-1.95版本）
    1) 在安装附带的某一先前的版本时，请务必在安装本版本前先安装该旧版本。
    2) 在有其它版本存在的环境下执行卸载时，请运行本版本的卸载程序。

5.2.3 接口连接

5.2.3.1 并行接口连接
    1) 我们已在LPT1接口上检验了该程序的性能。
    2) 请在BIOS的并行模式设置中将ECP或EPP设置为“Bi-Directional”。
    3) 根据所使用的BIOS版本，可能会出现ECP无效而EPP有效，或反之EPP无效而ECP
       有效的情况。如果上述任一设置无法正常生效，请修改上述设置并重试。
    4) 关闭设备后，在ServiceObject识别出电源已关闭前，会有几秒钟的延迟。
       在实际应用中敬请见谅。
    5) 当在SetupPOS中注册某设备并在64位OS环境中使用该设备时，如果该型号不使用
       “TMPORT”端口类型，Windows的标准并行驱动程序将支持该设备。
       在上述情况下，会有如下限制：
        - 不支持电源监控功能。
        - 不支持使用命令来清除可恢复的错误。通过电源OFF/ON来解除错误状态。
        - 在打印机处于错误状态时，无法打开CashDrawer。只有在解除错误状态后，
          才能打开CashDrawer。
        - 如果打印机（并行接口模式）与PC连接，会显示“发现新硬件”对话框与标题为
          “Windows需要为您的EPSON TM-P X.xx安装软件”对话框。
          由于无需安装该驱动程序，应使用“定位并安装驱动程序软件（推荐）”
          选项搜索该驱动程序。
          当Windows表示无法定位驱动程序时，点击“关闭”按钮。一旦操作后，
          则不再显示该消息。

         * 如果点击[取消]按钮关闭该对话框，则下次连接该设备时仍将显示同一消息。

5.2.3.2 USB连接
    1) 我们已通过USB接口连接多达10台打印机检验程序的性能。
    2) 当在SetupPOS中使用“TMPORT”注册两台或多台同一型号的设备时，
       请在打开该设备电源前在“TMPORT Making”对话框中创建USB TMPORT。
       如果在创建该端口前打开该设备电源，
       则该设备将有可能无法连接到所需的端口上。
    3) 如果打开/关闭打印机时显示“发现新硬件向导”，
       则取消该向导并安装EPSON高级打印机驱动程序（4.50版本或更高版本）。

5.2.3.3 网络连接
    1) 我们已通过网络接口连接多达20台打印机检验程序的性能。
    2) 在使用网络接口设备时，请注意以下各点。
        - 在Open或Claim过程中，支持TM打印机电源关闭/打开操作，但在电源打开后，
          不能保证可以重新连接，因此请谨慎操作。在Open或Claim过程中，PC会与TM
          打印机保持独占连接（其它PC无法与之连接）。但是，如果TM打印机的电源一
          旦被关闭再打开，则该网络上存在的任何PC都可与该TM打印机连接。此时，
          处于Claim状态的某PC的OPOS将尝试自动重新连接。
          但如果另一台PC的OPOS同样也在尝试发起Claim，
          则无法确定哪一台PC能够最终取得连接。最早Claim连接的PC可以建立连接。
          相应地，如果一台TM打印机在发出Open或Claim命令后被关闭并再打开电源，
          则不一定能够取得重新连接，因此请谨慎操作。当打开TM打印机的电源时。
        - 需要长达20秒钟的时间来打开或重置TM打印机，或使用某项网络功能。
          在开始任何操作前，请等待此间隔时间。

5.2.3.4 Bluetooth connection
    1) 在关闭打印机电源后20秒以内，即便重新打开电源也不能打印。

5.2.4 设备
    1) 多色阶打印：
        由于多色阶打印需要更多的打印数据，
        通过串行或Ethernet接口进行打印可能会导致打印中断，
        且打印输出可能会出现白色条纹。

5.2.5 OPOS应用程序

5.2.5.1 当使用HardTotals时
    1) 使用EPSON系列NVRAM的Hard Totals需要驱动程序以便访问NVRAM：
       EPSON NVRAM驱动程序。在未安装EPSON NVRAM驱动程序的环境下，无法安装Hard
       Totals。要将Hard Totals安装到您希望使用EPSON NVRAM驱动程序的机器上，
       请首先安装该驱动程序。有关如何购买EPSON NVRAM驱动程序，
       请与您购买IM硬件的公司联系。

5.2.5.2 当使用KeyLock时
    1) 如果您试图在Windows2000下使用该功能，则OPOS无法正确检测到KeyLock的位置。
       因此，DM-KX060（为ARCX(IM-700)/URBANII(SR-610的可选KeyLock）的KeyLock功
       能不可用。

5.2.5.3 当使用CheckScanner时
    1) 当扫描图像数据时请勿触摸CheckScanner设备。
       尤其是请勿在扫描期间关闭并打开电源。
       否则ServiceObject的运行可能会出现问题。

5.2.6 注意事项
    1) OS的待机/睡眠模式
       当打印机处于运行状态时，如果计算机进入待机/睡眠模式，
       则驱动程序有可能会变得不稳定，因此请避免使用待机/睡眠模式。

5.2.7 其它
    1) OPOS APG/CPG文件
       从1.7版本开始，由于OPOS文件已被并入UnifiedPOS文件，故不再单独提供OPOS
       文件。请参照UnifiedPOS文件。
       有关OPOS APG/CPG(UPOS)文件的信息，可通过以下链接获取：（2011年12月）
            http://www.nrf-arts.org/content/unifiedpos
    2）关于操作系统的升级
       如果在已安装了驱动程序的环境下进行操作系统的升级，则请在操作系统升级后重
       新登录硬件。
    3) 日志文件的保存位置
       跟踪日志文件与POSPrinter错误日志文件的默认保存位置如下所示：
        Windows Vista/7/8/Embedded POSReady 7/Embedded Standard 7：
          （系统驱动）：
            \Program Data\EPSON\OPOS
        Windows 2000/XP：
          （系统驱动）：
            \Documents and Settings\All Users\Application Data\EPSON\OPOS
        Windows Server：
          （系统驱动）：
            \Program Data\EPSON\OPOS
        Windows Embedded：
          （系统驱动）：
            \Documents and Settings\All Users\Application Data\EPSON\OPOS
    4) 使用OPOS后从打印机发送的数据
       一旦使用OPOS，则会覆盖设备设置。根据此操作，数据可能随后从设备中发出。
       如果未使用OPOS而直接连接到设备，则请忽略以下数据。
        - 当打开电源时，发送3个字节的打印机状态数据
        - 当打开电源时，发送3个字节的电源打开通知数据
    5) OPOS ADK 2.60版本卸载程序存在尚未解决的问题。
       因此，OPOS ADK 2.60版本卸载时，请在安装该版本后执行卸载。
    6) 在进行升级安装后卸载OPOS时，在安装文件夹内可能仍然存在遗留的空文件夹。如
       有必要，可以删除上述空文件夹。
       请注意，如果未删除上述空文件夹，也不会对驱动程序的运行产生任何影响。

6.2.3 接口连接

6.2.3.1 并行接口连接
    1) 我们已在LPT1接口上检验了该程序的性能。
    2) 请在BIOS的并行模式设置中将ECP或EPP设置为“Bi-Directional”。
    3) 根据所使用的BIOS版本，可能会出现ECP无效而EPP有效，或反之EPP无效而ECP
       有效的情况。如果上述任一设置无法正常生效，请修改上述设置并重试。
    4) 关闭设备后，在ServiceObject识别出电源已关闭前，会有几秒钟的延迟。
       在实际应用中敬请见谅。

6.2.3.2 USB连接
    1) 我们已通过USB接口连接多达10台打印机检验程序的性能。
    2) 当在SetupPOS中使用“TMPORT”注册两台或多台同一型号的设备时，
       请在打开该设备电源前在“TMPORT Making”对话框中创建USB TMPORT。
       如果在创建该端口前打开该设备电源，
       则该设备将有可能无法连接到所需的端口上。

6.2.3.3 网络连接
    1) 我们已通过网络接口连接多达20台打印机检验程序的性能。
    2) 在使用网络接口设备时，请注意以下各点。
        - 在Open或Claim过程中，支持TM打印机电源关闭/打开操作，但在电源打开后，
          不能保证可以重新连接，因此请谨慎操作。在Open或Claim过程中，PC会与TM
          打印机保持独占连接（其它PC无法与之连接）。但是，如果TM打印机的电源一
          旦被关闭再打开，则该网络上存在的任何PC都可与该TM打印机连接。此时，
          处于Claim状态的某PC的OPOS将尝试自动重新连接。
          但如果另一台PC的OPOS同样也在尝试发起Claim，
          则无法确定哪一台PC能够最终取得连接。最早Claim连接的PC可以建立连接。
          相应地，如果一台TM打印机在发出Open或Claim命令后被关闭并再打开电源，
          则不一定能够取得重新连接，因此请谨慎操作。当打开TM打印机的电源时。
        - 需要长达20秒钟的时间来打开或重置TM打印机，或使用某项网络功能。
          在开始任何操作前，请等待此间隔时间。

          则不一定能够取得重新连接，因此请谨慎操作。当打开TM打印机的电源时。
